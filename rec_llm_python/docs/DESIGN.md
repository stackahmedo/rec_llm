# RecLLM システム設計書 (Design Document)

## 1. システム概要

RecLLMは、長時間音声ファイル（最大30時間）の自動文字起こし、AI要約、話者分析を行うデスクトップアプリケーションです。

│  │  - Dashboard                          │  │

## 2. 処理パイプライン

### 2.1 音声処理フロー

```
音声ファイル入力
    │
    ▼
メタデータ取得 (FFprobe)
    │
    ▼
ティア判定
    ├── Normal (< 2h): 直接処理
    ├── Long Audio (2-10h): 45分チャンク, 並列2
    └── Enterprise (10-30h): 25分チャンク, 逐次処理
    │
    ▼
[オプション] ノイズ除去 (FFmpeg afftdn)
    │
    ▼
チャンク分割 (FFmpeg)
    │
    ▼
AssemblyAI アップロード + 文字起こし
    │ (リトライ: 最大3回, 指数バックオフ)
    ▼
ストリーミングマージ (メモリ効率)
    │
    ▼
SQLite保存 + FTS5インデックス
    │
    ▼
[オプション] AI要約 / 文法修正 / 翻訳
    │
    ▼
完了通知
```

### 2.2 ジョブキュー

- asyncioベースの非同期キュー
- 設定可能な並列度 (デフォルト: 2)
- クラッシュリカバリ: 起動時に`running`状態のジョブを`queued`にリセット
- 失敗ジョブは自動リトライしない（手動リトライ）

### 2.3 ストリーミングマージ

30時間音声のメモリ安全性を確保:
- チャンクごとに結果をマージ
- マージ後に個別結果をNullに設定（GC対象）
- タイムスタンプオフセット自動計算
- 失敗チャンクをスキップ（部分結果を保持）

## 3. セキュリティ

- APIキーはSQLiteに保存（ローカルのみ）
- ネットワーク通信はlocalhost限定
- 外部APIへの通信はHTTPS
- ファイルパスのサニタイズ
- SQLインジェクション防止（パラメータ化クエリ）

## 4. スケーラビリティ

| 指標 | 設計値 |
|------|--------|
| 最大音声長 | 30時間 |
| 最大ファイル数 | 1000+ |
| 最大テキスト量 | 2M+ 文字 |
| データベースサイズ | 1TB対応 (SQLite) |
| 同時処理 | 2ジョブ (設定可能) |
| FTS5検索 | 100万発話対応 |

## 5. エラーハンドリング

- チャンク失敗: 最大3回リトライ、指数バックオフ
- API接続失敗: タイムアウト後にエラー報告
- ディスク容量不足: 事前チェック
- クラッシュ: 起動時に孤立ジョブを回復
- 部分失敗: 成功チャンクの結果を保持
